# 출시 준비 (LAUNCH)

> 딱풀을 구글 플레이에 올리기까지의 실무 체크리스트.
> **1차 출시는 전면 무료**(결제 미탑재). 결제 연동(§3·§6)은 나중에 켤 때만 본다.

## 0. 무료 우선 출시 — 지금 경로 (⭐ 이걸 따르세요)

**결정: 처음엔 무료로 낸다.** 코드는 이미 이걸 지원한다 — `Monetization.LAUNCH_FREE_UNTIL_MILLIS`
(현재 KST 2026-11-01) 이전에는 `hasFullAccess`가 항상 true라 **난이도 1~10·심화풀이·전체 리포트가
모두 무료로 열린다.** 그래서 **실결제(§6)는 1차 출시의 차단 요소가 아니다.**

무료 출시 게이트 (하나라도 안 되면 배포 금지):
- [x] **릴리스 서명(업로드 키)**: `keystore.properties` + `ddakpul-upload.keystore` 생성 완료,
      `./gradlew :app:bundleRelease`로 **서명된 AAB 산출 확인**(11MB). ⚠️ 이 두 파일은 gitignore됨 —
      **사람이 안전한 곳에 백업**해야 함(잃으면 앱 업데이트 곤란).
- [x] **개인정보 안전**: 아동 식별정보 수집 0(무서버·온디바이스). 광고/애널리틱스 SDK 금지. (§4·§5)
- [x] **구매 UI 원천 차단**: `Monetization.BILLING_ENABLED=false` 동안 가격·활성화 버튼이 어디에도
      노출되지 않음(페이월은 무료 안내+혜택 소개만). 실결제 연동 PR에서만 true로 전환.
- [x] **네이티브 라이브러리 Play 정책 준수**: TTS용 .so를 런타임 다운로드하던 방식(정책 위반:
      실행 코드 외부 다운로드 금지)을 **빌드 시점 AAB 동봉**(해시 검증)으로 교체. 모델 파일(데이터)만
      런타임 다운로드 유지 + 다운로드 완결성 검증(Content-Length) 추가.
- [ ] **무료 창 마감일 조정**: `LAUNCH_FREE_UNTIL_MILLIS`를 실제 출시 계획에 맞춰 설정
      (무기한 무료면 아주 먼 미래로, 나중에 유료 전환 계획이면 그 시점으로).
- [ ] **사람이 할 일**: Play Console 등록($25) → 스토어 등록정보 → AAB 업로드 → 심사 (아래 §1·§2·§7·§8).

> 콘텐츠 규모: 현재 **963문항**(생성 830 + 카탈로그 133), 난이도 1~10, 한/영 이중언어. 무료 출시엔 충분.
> 나중에 유료 전환 시 유료 티어(난4+) 깊이·반복도만 다시 점검.

### 결제를 나중에 켤 때만 볼 게이트 (지금은 무시)
- [ ] 실결제 연동: 페이월이 로컬 활성화 대신 Play Billing을 호출하도록 교체. (§6)

## 1. Play Console 개발자 등록

1. https://play.google.com/console → **개발자 계정 등록**(개인/사업자). **등록비 $25(1회)**.
2. 신원 확인(D-U-N-S 또는 개인 신분증). 며칠 걸릴 수 있음.
3. 결제 프로필 설정(수익 수령 계좌).

## 2. 앱 만들기

1. Console → **앱 만들기**. 이름 "딱풀", 언어 한국어, **앱/무료 여부: 앱**, 유료 기능(인앱결제) 있음.
2. **패키지명 `com.ddakpul.math`** (코드와 일치).
3. **콘텐츠 등급**: 설문 작성(교육, 폭력/성적 콘텐츠 없음) → 전체이용가.
4. **타겟층·아동 정책(Google Play Families)**: 대상 연령에 어린이 포함 → **Families 정책** 적용. 아래 준수사항 자동 충족됨:
   - 제3자 광고·애널리틱스 없음 ✅ (넣지 말 것)
   - 개인식별정보·기기ID 전송 없음 ✅
   - 외부 링크·구매 유도 없음 ✅
   - **Teacher Approved** 심사 대상일 수 있음(키즈 탭 노출 원하면).

## 3. 인앱 상품 등록 (코드의 productId와 1:1)

Console → 수익 창출 → **인앱 상품**(관리형 상품, non-consumable/one-time). 코드는 기간제 이용권을 로컬 만료로 관리하므로 **일회성 상품**으로 등록:

| 상품 ID | 이름 | 기간(코드) | 제안 가격 |
|---|---|---|---|
| `ddakpul_pass_6m` | 6개월 이용권 | 182일 | ₩9,900 |
| `ddakpul_pass_1y` | 1년 이용권 | 365일 | ₩19,000 |

- productId는 `PremiumPass` enum(`domain/model/Entitlement.kt`)과 반드시 일치.
- **자동갱신 없음** — 구독이 아니라 일회성. 한국 '구독 해지방해' 규제 회피 + 솔로 개발자 유지부담↓.
- 가격은 스토어에서 받아 `PremiumPass.defaultPriceLabel`을 대체(§6에서 연동).

## 4. 데이터 안전(Data Safety) 폼 — 가장 쉬운 케이스

Console → 앱 콘텐츠 → **데이터 안전**. 딱풀은 무서버·온디바이스라 답이 전부 "아니요":

- 데이터를 수집하거나 공유하나요? → **아니요**
- (수집 안 하므로 유형·목적 항목 없음)
- 전송 중 암호화? → 해당 없음(전송 자체가 없음)
- 사용자가 데이터 삭제 요청 가능? → 앱 내 '학습 기록 초기화'로 전부 삭제 가능
- **개인정보처리방침 URL**: 필수. §5 텍스트를 어딘가(깃허브 Pages 등)에 올리고 그 URL 기입.

## 6. 실결제(Play Billing) 연동 — Console 상품 등록 후 개발 작업

Console 상품(`ddakpul_pass_6m`·`ddakpul_pass_1y`)이 준비되면:

1. `com.android.billingclient:billing-ktx` 의존성 추가.
2. `BillingRepository`(신설) + `PlayBillingRepositoryImpl`: BillingClient 연결 → `queryProductDetails` → `launchBillingFlow(activity, ...)` → 구매 완료 리스너에서 `acknowledgePurchase` → **검증 성공 시 `ActivatePassUseCase(pass, now)` 호출**.
3. `PaywallViewModel.activate`가 로컬 활성화 대신 `BillingRepository.purchase(activity, pass)`를 호출하도록 교체. Activity 핸들을 넘겨야 하므로 페이월에서 `LocalActivity`를 받아 전달.
4. 상품 가격을 `queryProductDetails` 결과로 표시(현재 `defaultPriceLabel` 대체).
5. **복원(restore)**: `queryPurchasesAsync`로 기존 구매 확인 → 재설치·기기변경 시 이용권 복구. (온디바이스 만료 모델이라, 복원 시 Play 구매 시각 기준으로 만료 재계산.)
6. **테스트**: Console '라이선스 테스터'에 계정 추가 → 내부 테스트 트랙에 서명 AAB 업로드 → 실결제 없이 구매 흐름 테스트.

## 7. 스토어 등록정보 (초안 — GTM 포지셔닝 기반)

**앱 이름**: 딱풀 — 초등 사고력 수학

**짧은 설명(80자)**:
광고 없는 진짜 사고력 수학. 아이 스스로 풀고, 부모는 성장 리포트로.

**자세한 설명**:
```
딱풀은 '딱 맞는 문제를 푸는' 초등 사고력 수학 앱이에요.

■ 계산이 아니라 사고력
연습장 펴고 5분쯤 고민하는 다단계 문제(퍼즐·추론·전략). 외우지 말고 스스로 깨닫습니다.

■ 광고 없이, 안전하게
광고도, 결제 유도도 없어요. 학습 기록은 이 기기 안에만 남고, 인터넷 없이도 동작합니다.

■ 아이 스스로, 부모는 리포트로
아이가 문제를 풀면 실력에 맞춰 다음 난이도를 자동으로 조절해요(난이도 1~5, 학년 구분 없음).
부모님은 성장 리포트로 어디까지 왔는지 한눈에 봅니다.

■ 보상 슬롯머신이 아니에요
뱃지·포인트로 유혹하지 않아요. 결과가 아니라 '과정'을 칭찬하도록 설계했습니다.

무료로 난이도 1~3까지 충분히 써 보고, 마음에 들면 이용권으로 4·5단계와 전체 리포트를 여세요.
자동 갱신 없는 6개월·1년 이용권 — 기간이 끝나면 그냥 멈춥니다.
```

**그래픽 자산 필요**: 앱 아이콘(512px), 피처 그래픽(1024×500), 스크린샷(태블릿·폰 각 2~8장).

## 8. 배포 순서 (권장)

1. 내부 테스트 트랙에 서명 AAB 업로드 → 라이선스 테스터로 결제·온보딩·게이트 전 흐름 점검.
2. 비공개 테스트(지인 몇 명) → 실기기 다양성·난이도 체감 확인.
3. 프로덕션 → 단계적 출시(%).
