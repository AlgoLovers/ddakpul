# 문제 데이터 소싱 전략 (CONTENT SOURCING)

> "연습장에 5분쯤 풀어야 하는 다단계 사고력 문제"를 저작권 리스크 없이 대량 확보하는 방법.
> 2026-07 라이선스 원문 기준 조사. 결론: **턴키로 사 올 수 있는 안전한 대량 소스는 없다.**
> PD 원전 각색(뼈대) + 템플릿·AI 생성 후 솔버 검증(물량) + CC BY 소스(보강)의 3축이 유일한 경로.

## 1. 바로 쓸 수 있는 소스 (라이선스 안전)

### A. 퍼블릭 도메인 퍼즐 원전 (약 500~800문제 각색 가능)

근거: 저자 사망 후 70년 경과(한국·영국·EU), 미국은 1930년 이전 출판물 PD.

| 책 | 링크 | 확보량 |
|---|---|---|
| Dudeney, *Amusements in Mathematics* (1917) | gutenberg.org/ebooks/16713 | 430문제, 30~40% 초등 각색 가능 — **1순위** |
| Dudeney, *The Canterbury Puzzles* (1907) | gutenberg.org/ebooks/27635 | 114문제, 스토리텔링형 |
| Sam Loyd, *Cyclopedia of 5000 Puzzles* (1914) | archive.org + Wikisource 전사본 | 수천 개(말장난류 제외 선별) |
| Carroll, *A Tangled Tale* (1885) | gutenberg.org/ebooks/29042 | 10화 이야기형 산술 |
| *The Magician's Own Book* (1857) | gutenberg.org/ebooks/60687 | 숫자 마술 수십 개 |

각색 원칙: 수치·인물·상황을 한국 맥락으로 전면 재작성(파운드→원, 스토리 교체).
재작성 자체가 2차적저작물로서 자체 저작권 발생. 문제마다 원전 출처를 기록한다.

⚠️ PD 함정 판본: Martin Gardner 편집본(©1959/1967), Kordemsky(1999†), Perelman Mir 영역판,
Dudeney 유고작 *Puzzles and Curious Problems*(1931, 미국 2027-01까지 비PD)는 쓰면 안 됨.

### B. CC BY 4.0 (상업·번역·앱 내장 허용, 출처표시 필수)

- **youcubed(스탠퍼드)** — 약관에 CC BY 4.0 명시. 일부 과제가 제3자 출처라 과제별 확인.
- **Illustrative Mathematics K-5 1st edition(2019–2021)** — CC BY 4.0.
  ⚠️ 신판 v.360(2024~)은 CC BY-**NC**라 불가. 판본 구분 필수.
- CC BY 문항용 **앱 내 크레딧 화면** 필요.

### C. 자체 오리지널 생성 (가장 큰 공급원, 무제한)

마방진·복면산·닭토끼 같은 **유형·아이디어는 저작권 보호 대상이 아니다**
(미 저작권청 Circular 33: 아이디어·방법·시스템 비보호).
특정 출판물의 문장·수치·스토리만 안 베끼면 무제한 생산 가능.

### D. 공공누리 제1유형 (물량 적음, 선별)

제1유형만 상업+변형 허용. 서울시교육청 홈페이지 저작물 등 — 문서 개별 마크 확인 필수.

## 2. 쓸 수 없는 소스 (흔한 착각 주의)

- **경시 기출 전부**: Math Kangaroo(조직위 계약 필요), MATHCOUNTS(재출판 금지 명시),
  AMC/MAA(영리 이용 서면 허락), MOEMS·CEMC(All rights reserved).
- **NC(비상업) 조항**: Open Middle, NRICH(수정·번역까지 금지), Khan Academy, EngageNY/Eureka.
- **한국 공공 데이터 대부분**: 에듀넷 평가문항 데이터셋은 공공누리 **제4유형**(상업·변형 금지).
  AI허브 수학 문항 3만+개는 모델 학습만 가능 — **원문 앱 내장은 재배포 금지 위반**.

## 3. 확보 파이프라인 (단계별)

1. **유형 그리드 확정** — 아래 유형 카탈로그 × 난이도 1~5로 출제 그리드, 칸별 목표 문항 수 배정
2. **PD 원전 수확** — Dudeney 2권 + Loyd 선별 → 한국 맥락 재작성 → 원전 출처 기록
3. **템플릿+AI 생성 (핵심)** — 유형별 파라미터 템플릿("민준이는 {x}개…" + 제약조건 + 풀이 코드)
   → LLM이 스토리·변형 생성 → **정답·유일해를 프로그램으로 검증**
   (복면산: cryptator / 논리 격자: Z3·grilops / 문장제: SymPy + 유일성 검사 "두 번째 해 발견 시 폐기")
   선례: DeepMind mathematics_dataset(Apache-2.0), GSM-Symbolic, Duolingo AIG
4. **인간 검수 + 캘리브레이션** — 솔버 통과분만 표본 검수(오개념·문화 적합성),
   실사용 정답률로 난이도 보정. 사람의 편집·선별 기록은 저작권 등록 요건도 충족
5. **출처 대장(provenance)** — 모든 문항에 {출처, 라이선스, 각색 이력} 필드 (JSON 에셋화와 함께)

**예상 총량**: PD 각색 500+ / 자체 생성 1,500~4,000+ / CC BY 수백 → **초기 2,000문항 현실적.**

## 4. 사고력 유형 카탈로그 (오리지널 제작용 43유형)

**수·연산 퍼즐**: 마방진(2–4) · 복면산(3–5) · 벌레먹은셈(2–4) · 숫자 카드 최대최소(1–2) ·
연산 기호 넣기(1–3) · 수 피라미드(2–4) · 자릿수 추리(3–4) · 약수배수·나머지(3–5) ·
홀짝성 논증(4–5) · 약속 연산(3–4)

**문장제(고전 산술)**: 닭토끼(3–4) · 나무 심기(2–3) · 과부족(3–4) · 나이(3–4) · 만남추격(4–5) ·
강물 배(5) · 기차 통과(5) · 시곗바늘 각도(4–5) · 달력요일(2–4) · 작업량(4–5) · 평균 변화(4) ·
소거법 묶음 비교(3–4) · 전후 비교(4–5)

**도형·공간**: 도형 개수 세기(2–4) · 성냥개비(1–3) · 도형 분할합성(2–4) · 쌓기나무(2–5) ·
전개도주사위(3–4) · 종이 접어 자르기(3–4) · 대칭회전(2–3) · 둘레넓이 재구성(4–5) ·
한붓그리기(3–4) · 격자 최단 경로(4–5)

**규칙성**: 수열 규칙(2–5) · 그림 패턴 n단계(2–4) · 반복 주기(2–3)

**논리·이산·자료**: 논리 격자(2–4) · 진실거짓(3–5) · 강 건너기·저울(3–5) · 님 게임 필승(4–5) ·
비둘기집(4–5) · 경우의 수 나열(2–5) · 표그래프 해석(2–3)

난이도 규칙(캥거루식): 1–2 = 기본 적용 / 3 = 두 단계 결합 / 4–5 = 비정형 서술 + 숨은 구조 + 다중 조건.

## 5. 큐레이션 (관리자 검수 흐름 — GitHub 기반)

문제 배치가 생성될 때마다 **GitHub 이슈에 문항별 체크리스트**가 올라온다.

1. 관리자(GitHub 로그인)가 이슈에서 **제거할 문제에 체크** ✅ — 웹/모바일 앱 어디서든
2. Claude에게 "검수 반영해줘"라고 지시(텔레그램/터미널)하면:
   - 체크된 id가 `tools/problemgen/blocklist.txt`에 영구 등록되고
   - 에셋을 재생성(블록리스트 자동 제외) → PR → 머지 → 앱에서 사라짐
3. 블록리스트는 누적된다 — 한 번 제거된 문제는 이후 어떤 재생성에서도 돌아오지 않는다
4. 손으로 만든 카탈로그 문항 제거는 Claude에게 id로 지시하면 코드에서 직접 삭제

## 6. 도형 문제 (figure DSL)

이미지 파일 대신 **도형 지시서**(타입+파라미터)를 문제에 붙이고 앱이 Canvas로 직접 그린다
— 해상도 무관, 용량 0, 화면과 인쇄(PDF) 동일 렌더링.

현재 타입: `CLOCK`(시계면) · `DOT_BORDER`(바둑돌 테두리) · `GRID`(격자/최단경로) · `L_SHAPE`(L자 도형).
새 유형이 필요하면 FigureType에 타입 추가 + 화면/PDF 렌더러 두 곳에 그리기 코드를 더한다.
