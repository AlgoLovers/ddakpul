## 0. 한눈에 (제일 중요한 5가지)

1. **가장 큰 함정 = 신규 개인 계정의 "폐쇄 테스트 12명 · 14일 연속" 게이트.** 이걸 통과해야 프로덕션에 낼 수 있다. 사업자(조직) 계정은 면제. → **계정 유형을 뭘로 할지가 전체 일정을 좌우한다.**
2. **첫 출시는 빨라도 2~3주 리드타임.** 신원확인(수일~수주) + 폐쇄테스트 14일 + 심사 1~7일. 일정 역산 필수.
3. **딱풀은 정책 리스크가 매우 낮다.** 데이터0·광고0·추적SDK0·오프라인이라 아동앱 정책에 오히려 유리. 최대 관리 포인트는 **"데이터 안전 폼 ↔ 개인정보처리방침 ↔ 콘텐츠 등급" 3중 일관성** 하나뿐.
4. **스토어 제목·설명에 "무료 / 광고없음 / 1위" 표기 금지**(구글 정책). 딱풀 최대 강점을 제목에 못 쓴다 → 스크린샷·본문·리뷰·맘카페로 우회 전달.
5. **이미 상당 부분 준비됨** — 서명 업로드키·targetSdk 36·스토어 그래픽·스크린샷·개인정보처리방침(라이브 확인 완료)까지 있다. 남은 건 대부분 "사람이 콘솔에서 클릭"하는 일. (§6 현황표)

## 1. ⏱️ 계정 유형 & 타임라인 — 가장 큰 함정

### 개인(Personal) vs 조직(Organization) — 먼저 결정
| | 개인 계정 | 조직 계정 |
|---|---|---|
| 신원확인 | 정부발급 신분증 | **D-U-N-S 번호 필수**(발급 지연 흔함, 무료) |
| **폐쇄 테스트 게이트** | **적용**(12명·14일) | **면제**(프로덕션 직행 가능) |
| 적합 대상 | 취미·개인 개발자 | 상업·기관 활동 |
| 스토어 표기 | 개인 실명·주소 노출 | 사업자명 노출 |

- 딱풀은 금융/건강/VPN이 아니라 **개인 계정으로 낼 수 있다**. 결정 포인트: **12명 게이트를 감수(개인)** vs **사업자 등록+D-U-N-S로 게이트 회피(조직)**.
  - 근거: 계정 유형·의무 — support.google.com/googleplay/android-developer/answer/13634885
- **추천**: 사업자 계획이 없으면 개인 계정 + 12명 게이트를 감수하는 게 가장 빠르다. 대신 **테스터 12명 명단을 지금부터 확보**(가족·지인).

### 폐쇄 테스트 게이트 (2023-11-13 이후 생성 개인 계정)
- **"20명"이 아니라 2024-12부터 12명으로 완화됨.** 정확히: **12명 이상이 14일 연속 opt-in 유지** → "프로덕션 액세스 신청" 승인 → 정식 출시.
  - 근거: support.google.com/googleplay/android-developer/answer/14151465
- **"opt-in"의 함정**: 이메일 초대만으론 카운트 안 됨. 각 테스터가 **초대 수락 + 본인 구글계정으로 실제 설치**해야 1명. 14일 중 인원이 12명 밑으로 떨어지면 **연속 기간이 리셋**될 수 있다. → 실계정 12개를 미리 섭외.

### 등록비 · 신원확인
- **등록비 $25(1회, 평생, 환불불가)**. 선불카드 불가. — answer/6112435
- **앱 제출 전에 신원확인 완료 필수.** 개인은 신분증+법적 이름/주소(검증 후 스토어 노출). 처리 소요 공식 명시 없음 → **수일~수주 대비**. — answer/10841920

### 리드타임 역산
- 개인 계정 실질 최소 ≈ **16~18일**(폐쇄테스트 14 + 프로덕션 액세스 심사 1~3 + 앱 심사 1~2). 신규 계정 첫 심사는 신뢰이력이 없어 **더 오래(최대 14일+)** 걸릴 수 있다. → **최소 2~3주 여유** 잡기.
  - 근거: ptkd.com/journal/google-play-review-time, capgo.app/blog/first-time-app-review-guide

## 3. 👶 아동 대상 앱 정책 (딱풀 = 유아 포함 → 필수)

> 과거의 별도 "Designed for Families 신청"은 사라졌고, **대상 연령에 아동을 선언하면 Families 정책이 자동 전면 적용**된다.

### 3중 일관성 — 이것 하나만 확실히
아동앱 최다 거부·정지 사유는 **신고 불일치**다. 아래 셋이 서로 모순되면 업데이트 차단·삭제.
1. **데이터 안전(Data Safety) 폼**: 딱풀은 수집=아니오 / 공유=아니오. (무수집 앱도 폼 제출 + 방침 링크 필수) — answer/10787469
2. **개인정보처리방침**: 실제 동작을 정확히 반영(이미 라이브·IP 고지 포함).
3. **콘텐츠 등급(IARC 설문)**: 폭력·성적·도박 등 "없음"으로 정직히 답 → 교육 퍼즐앱은 자연히 **전체이용가(3+)** 산출. — answer/9859655

### "데이터 수집"의 정의 = 기기 밖 전송
- 내가 받든 제3자가 받든, **앱이 쓰는 라이브러리/SDK가 보내는 것까지** 전부 "수집"으로 신고 대상. 온디바이스 처리·일시적(ephemeral) 처리는 공개 신고 제외. — answer/10787469

### ⚠️ 선택적 TTS/영상 다운로드 & IP 전달 (유일한 회색지대)
- 파일을 GitHub/Hugging Face에서 받는 순간 **기기 IP가 제3자 호스트에 전달**된다. 원칙상 "기기 밖 전송"에 해당하나, IP를 그 요청 처리에만 쓰고 저장 안 하면 **ephemeral 예외**로 공개 Data Safety 섹션엔 미표시 가능.
- **딱풀은 이미 개인정보처리방침에 "선택 다운로드 시 제3자 호스트에 IP 전달" 사실을 공개**해 둠 → 가장 안전한 형태. (보수적으로 가려면 다운로드를 보호자가 시작하는 명시적 선택 기능으로 유지 + 그 라이브러리에 광고ID/식별자 부가 기능이 없는지 코드 점검.)
- **최종 판단만 출시 직전 한 번 더**: (A) ephemeral로 보고 "수집없음" 유지 + 방침 공개(권장·현재 상태) / (B) 보수적으로 "앱 기능" 목적 최소 신고. 어느 쪽이든 **셋이 모순 없을 것**.
  - 근거: answer/10787469, Families — answer/9893335

### 아동앱 추가 준수(딱풀은 이미 충족)
- 개인화/추적 광고 금지, 미승인 SDK 금지, 기기식별자(AAID·IMEI·MAC…) 전송 금지 → 딱풀은 광고·추적 SDK가 없어 자동 준수. — answer/9893335
- **한국법**: 만 14세 미만 개인정보 수집 시 법정대리인 동의 필수 → 딱풀은 "수집 0"으로 원천 회피.

### 🎁 기회: Teacher Approved (키즈 탭 노출)
- Families 정책을 충족하면 **자동으로 교사 심사 대상**(별도 신청서 없음). 통과 시 **플레이 키즈 탭 노출 + "Teacher Approved" 배지 + 학부모 발견성↑**.
- 평가 7기준: 언어/오디오·UI 직관성·비주얼·몰입·학습촉진·콘텐츠 적합성·**광고/결제/상호홍보 적절성**. 딱풀의 무광고·사고력 특성과 궁합 좋음. **단 "유아~성인"으로 넓게 선언하면 가장 어린 연령 기준까지 통과**해야 함.
  - 근거: play.google.com/console/about/teacherapproved

## 4. 🔍 시장 · ASO · 마케팅 팁

### A. 메타데이터(검색 노출의 핵심)
- **제목 30자·핵심 키워드 1개.** 신생 앱은 브랜드보다 기능 키워드를 앞에. 스터핑 금지.
- **제목·짧은설명에 "무료 / No Ads / #1 / 베스트" 금지**(혜택·순위 표현 정책 위반). → 딱풀 강점은 **스크린샷·설명 본문·리뷰·맘카페**로 전달.
- **짧은설명(80자)**에 핵심 키워드 1회, **긴설명(4,000자)**은 ~250자당 키워드 1회·앞문장에 승부("5분 생각하는 사고력 문제", "정답에 따라 다음 문제 자동 추천").
  - 근거: support.google.com/googleplay/android-developer/answer/13393723

### B. 비주얼 에셋
- **피처그래픽 1024×500 = 시각 초점 1개 + 짧은 헤드라인 + 강한 색.** 기능 나열·가장자리 중요요소 배치 금지.
- **스크린샷은 '기능 덤프'가 아니라 이점을 파는 스토리.** 각 장면이 하나의 이점을 증명. **태블릿 우선이니 태블릿/가로 스크린샷 반드시 별도 등록**. (`/emu-qa`로 라이트/다크 캡처)
- 금지: 타사 로고/캐릭터, 안드로이드 기기 이미지, Play 배지, 수상·순위 문구.

### C. 최신 랭킹 흐름(2025~2026)
- 키워드뿐 아니라 **앱 품질 신호(크래시 없음·업데이트 빈도·리텐션·평점)**가 순위에 크게 반영. ASO는 1회성이 아니라 상시 최적화.

### D. 초기 리뷰·설치 (합법 범위)
- **인센티브성 리뷰(현금·할인·경품·기부 대가) 전면 금지** → 계정 정지 리스크. 리뷰 매수·체험단 대가 리뷰 절대 금지.
- **합법: In-App Review API를 만족 정점에** (리포트 확인·난이도 승급 직후). **"맘에 드세요?/5점 주세요" 유도 질문은 구글 명시 금지.**
- **사전 등록(Pre-registration)**으로 출시 첫날 설치 몰기. **메타데이터·에셋은 출시 전에 완성**(첫 주 부스트 낭비 방지).
  - 근거: developer.android.com/guide/playcore/in-app-review, answer/9859047

### E. 출시 전략
- **첫 출시엔 %단계 롤아웃 불가**(staged rollout은 업데이트 전용). 첫 프로덕션은 국가 한정/테스트 트랙으로 소프트런치.
- **업데이트는 1→5→10→…→100% 단계 + 각 단계 24h 관찰**, 문제 시 halt.
- **스토어 등록정보 실험(무료 A/B)**로 아이콘·스크린샷·설명 검증: 한 번에 한 요소·최소 14일·신뢰도 90%+. (제목은 실험 불가, 업데이트로만)

### F. 한국 시장 · 무료+광고없음 포지셔닝
- **강점**: 한국 교육시장은 '가격'보다 '신뢰'로 승부 — **광고없음·아동데이터0**은 강력한 신뢰 신호. 구매 결정자는 **학부모** → 문구는 학생이 아니라 학부모(리포트·안전·사고력)를 겨냥.
- **약점**: 무료는 초기 예산 부족 + '무료=저품질' 인식 + 제목에 '무료' 표기 불가 → 리뷰·커뮤니티 신뢰로 우회.
- **수익화 타이밍**: 무료로 신뢰·볼륨 확보 후 유료 재도입이 정석(딱풀 방침과 일치). 결제 미연동 중 가격·구매 버튼 노출 금지(헌법 원칙 7).

### G. 맘카페·커뮤니티 홍보 — ⚠️ 법적 주의
- **뒷광고(내돈내산 가장·자문자답 허위후기)는 형사 입건 전례**(180여 맘카페, 매출 70억). 협찬/제공 시 **'광고·협찬' 명시 필수**.
- **과장광고 금지(표시광고법)**: "○○하면 수학 1등", "IQ 상승" 같은 **효과 단정 금지** — 근거 있는 표현만.

### 추천 진행 순서
1. **계정 유형 결정** → 등록($25) → 신원확인 착수(가장 오래 걸림, 먼저).
2. (개인 계정이면) **테스터 12명 섭외** 동시 진행.
3. 콘솔 앱 생성 → **대상연령·데이터안전·콘텐츠등급 3종을 모순 없이** 작성 → 개인정보처리방침 URL 등록.
4. 스토어 등록정보 문구·그래픽 채우기(메타데이터는 출시 전 완성).
5. 서명 AAB를 **내부 테스트 → 폐쇄 테스트(12명·14일)** 업로드.
6. 프로덕션 액세스 신청 승인 → **국가 한정 소프트런치**로 프로덕션.
7. 출시 후 In-App Review·스토어 실험·평점 관리 상시 운영, Teacher Approved 노림.
