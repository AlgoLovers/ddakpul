## 2. 축별 전략

### 2.1 추천 알고리즘 — 규칙 계단에서 Elo-lite + 85% 규칙으로

**현황(이미 됨).** 8규칙 전부 구현·테스트 완료(`RecommendNextProblemUseCase`). 우선순위 `4정체 &gt; 8복습 &gt; 1·2승강급 &gt; 7재도전 &gt; 3유지`. 그룹 단위·순수함수·온디바이스·런타임 AI 0. 인터리빙·무료 상한 게이트·제외 필터까지 있음.

**근거.**
- **85% 규칙**(Wilson et al. 2019, *Nature Comms*): 오답률 ~15.87%(정답률 85%)에서 학습 속도 최대. → 문항 선택 목표이자 승강급 신호로 동시 사용.
- **Elo-lite**(Pelánek 2016; Duolingo Birdbrain): 학습자 θ·문항 b를 매 응답마다 정수 갱신, `R_new = R_old + K·(정답 − 기대정답률)`, K는 시도 누적 시 감소(Glicko RD 근사). 부동소수 없이 룩업테이블로 온디바이스 완결.
- **완전학습**(Bloom 1968)·**ZPD**(Vygotsky)·**바람직한 어려움**(Bjork): 승급은 스트릭이 아닌 **숙달 게이트**로, 선택은 θ보다 살짝 위 밴드로, 영역 교차로 유창성 착시 방지.
- **간격반복**: Leitner(현재)에 그룹별 정수 **ease factor**(SM-2식)를 더해 FSRS의 핵심 이득(불필요 복습 감소)을 근사.

**적용안 (온디바이스·정수 연산 유지).**
1. `LearnerState`에 **영역별(또는 개념별) θ 레이팅** 추가(정수). 문항 b는 난이도(1~10)→레이팅 초기치 + 응답으로 미세조정.
2. 승강급을 "2정답/2오답 스트릭" → **최근 창 정답률 + θ 마진**으로 대체(노이즈에 강함, 8번째 공백=10시도 창 한계 완화).
3. 문항 선택을 **기대정답률 0.85 밴드**(b ≈ θ − 약 300)에서 미출제 문항 랜덤 → ZPD 위치 정량화.
4. **진단↔처방 연결**: 영역별 θ가 생기면, 리포트가 짚는 취약 영역으로 추천이 실제 유도(현재 `areaMastery`가 추천에 안 쓰이는 공백 #1 해소).

**주의.** ROADMAP §6.1 Elo-lite·§6.2 BKT-lite가 이미 후보로 등록됨. 이 연구는 그 결정을 **확증**한다. 전면 IRT/FSRS는 과하다 — 정수 근사가 정답. (Khan Academy가 Elo를 쓴다는 통설은 근거 약함 → Duolingo·학술 시스템으로 귀속.)

### 2.2 문제 품질 — DOK3+ / 낮은문턱·높은천장 + 11항 감사 루브릭

**현황(이미 됨).** "계산 드릴 금지 = 사고력만"은 헌법. 브루트포스 솔버가 정답 유일성 검증. `problem-auditor` 에이전트가 콘텐츠 감사.

**근거.**
- **Pólya(1945)** "계획 세우기가 비자명" = 딱풀의 "연습장 5분"의 조작적 정의.
- **Webb DOK(2002)** DOK3+(전략적 사고) / **Stein &amp; Smith(1998)** 고수준 과제만 복합 사고 유발. "계산 드릴"=저수준(연결 없는 절차).
- **낮은문턱·높은천장**(NRICH, Boaler): 5세~성인 단일 축에 필수 — 진입은 쉽고 추론은 깊게.
- **Leikin(2009)** 다중해법으로 창의성(유창성·유연성·독창성).
- 한국 **KEDI 영재교육** 창의적 문제해결력·수준 I/II/III 단계 구조가 1~10 축과 호환.

**적용안 — 11항 pass/fail 감사 루브릭(→ `problem-auditor` 체크리스트로 이식).**
_문제:_ ①다단계성(≥2추론) ②DOK3+/고수준 ③유일 정답 ④낮은 문턱 ⑤오답 진단력(오답 3개가 각각 식별 가능한 오개념, 중복 없음) ⑥연령·난이도 적합.
_해설:_ ⑦'왜'의 설명(자기설명 대리) ⑧단계 구조+페이딩 ⑨오답별 교정 ⑩표상 다양성(기호 외 1개↑) ⑪되돌아보기(검산·일반화·다른 풀이).

### 2.3 풀이 품질 — 예제효과에서 페이딩·자기설명·다중표상으로

**근거.**
- **예제효과**(Sweller): 초보는 완전 해설 예제로 인지부하↓. 딱풀 상세해설이 이미 구현.
- **전문성 역전**(Kalyuga 2007): 숙련자에겐 완전 예제가 잉여 → **점진적 소거 예제(faded)**. **1~10 난이도 축을 페이딩 스케줄로**: 저층=완전 단계해설, 고층=힌트만.
- **자기설명 유도**(Chi 1989/1994): 해설을 "무엇을 했나" 나열이 아니라 각 단계 **"왜 이 선택이 정당한가"**로.
- **다중표상**(Lesh 1987): 해설에 최소 1개 비기호 표상(그림·표·도식). ROADMAP v0.3 "해설 수직선·그림 표상"과 직결.

**적용안.** 해설 저작 규칙을 루브릭 ⑦~⑪로 표준화. 페이딩은 난이도 층에 연동(콘텐츠 생성 시 detail 수준을 난이도 함수로).

### 2.4 통계 화면 — 성장지향으로 재설계 (가장 민감한 결정)

**현황(이미 됨).** `LearningStats` 단일 원전, 홈+부모 리포트 공유. 13개 지표 구현. 이미 확정된 원칙: 과정 칭찬·관대한 스트릭·리더보드 없음·부모팁(수학불안 경고 포함)·숙달을 점수 아닌 **단계 라벨**.

**근거.**
- **SDT**(Ryan &amp; Deci 2000): 같은 지표도 정보적이면 유능감↑, 통제적이면 내적 동기 잠식.
- **성장 vs 고정 마인드셋**(Mueller &amp; Dweck 1998, *JPSP*): 능력/순위 숫자 = "지능 칭찬" 효과(도전 회피). 과정/성장 숫자 = "노력 칭찬" 효과.
- **게이미피케이션 역효과**(Hanus &amp; Fox 2015): 리더보드·배지가 16주에 걸쳐 내적 동기↓.
- **부모 불안 전이**(Maloney et al. 2015, *Psych Science*): 잘못된 리포트가 불안 전달 통로. 처방 = 순위가 아니라 "어디 막혔고 부모가 뭘 하면 되는지".

**적용안 — 현재 6지표 비평·조정 (변경은 사용자 결정 필요).**
| 지표 | 판정 | 조정안 |
|---|---|---|
| 난이도별 숙달지도 | ✅ 최강 자산 | Khan식 단계 색 유지. 빈칸=결핍 아닌 "미개척지"로 프레이밍 |
| 개념 숙달도 | ✅ 유지 | "미달=빨강" → "여정의 다음 칸" |
| 오늘의 목표 | ✅ 유지 | 자율성 강화(조절 가능), 미달 죄책감 신호 금지 |
| 평균 풀이 시간 | ⚠️ 위험 | 학습자 화면에선 "숙고 시간"으로 중립화 또는 제거, 보호자만 참고. "빠름=좋음"은 사고력 철학과 상충 |
| 연속 학습일(streak) | ⚠️ 리셋형 | 누적·회복형("이번 달 학습 12일")으로, 또는 유예 기본 |
| 정답률 | 🔴 재고 | 수행지향 대표 허영지표+부모 불안 방아쇠. **내부 추천엔 계속 쓰되 전면 표시는 성장지표(숙달로 전환된 개념 수)로 대체** |

**대시보드 지침.** 학습자=자기대비 성장·숙달지도·도달난이도·극복서사·조절가능 목표. 보호자=개념별 숙달/막힘 + **행동 제안** + 꾸준함 격려. 둘 다 피할 것=리더보드·리셋 streak·정답률 전면·또래 백분위·속도 순위.

### 2.5 문제 수 확대 — 품질 게이트를 건 규모화

**현황.** 830문항, 209방법, 4영역 균형(196/219/210/205). 난이도 분포 `{4:190, 5:170}`에 무게중심(43%), **천장(10)은 32로 얇음**.

**적용안.**
- **품질을 규모의 전제로**: §2.2의 11항 루브릭 + 솔버 검증을 통과 못 하면 추가 금지. 규모≠양산.
- **커버리지 리포트로 빈틈 조준**: 영역×난이도 슬롯 커버리지를 자동 산출(§2.6-D), 얇은 슬롯(상위 난이도) 우선 채움. "천장 한 칸=4영역 모두" 헌법 준수.
- **템플릿 추상화로 함수 폭발 없이 확장**(§2.6-E): 새 변형=파라미터 선언, 문장/오답/그림/이중언어 상속.
- 수제 카탈로그 133문항 코드 미부여분 처리(다음 배치).

### 2.6 생성 파이프라인 재구성 — 모놀리스에서 패키지로 (핵심 인프라)

**결함·병목(실측).**
- `generate.py` 6097줄 = 전체 89%. **동일 함수 중복 정의 3건**(`gen_consecutive_sum`·`gen_handshake`·`gen_pigeonhole`)이 조용히 덮어써짐 — 파일이 커질수록 못 잡는다.
- **`code` 필드 재생성 부재**: 커밋 3ebefbe 일회성 주입, 주입기 미커밋. `generate.py` 통째 재실행 시 **830코드 전부 소실**. (이번 난이도 수정도 이 때문에 JSON 직접 편집으로 우회함.)
- 수동 `GENERATORS` 리스트 이중 관리, 템플릿 재사용성 0(조사교정·오답·이중언어 매 함수 복붙).

**재구성안 (앱 런타임 무손상 — 데이터 파일만 갱신, Kotlin 불변).**
- **A. 패키지 분할**: `core/`(emit·조사교정·distractor·figure) + `generators/`(영역별 4파일) + `build.py`(레지스트리→게이트→코드주입→저장).
- **B. 레지스트리**: `@register` 데코레이터로 임포트 시 자동 등록, 중복 이름을 등록 시점 에러로.
- **C. 코드주입 자동화 (필수·회귀 방지)**: `build.py`가 `method_codes.json`+`groupId`로 `AA-BB-CC-DD-SS`를 항상 재구축. 신규 방법이면 매핑 자동 갱신, 미분류(99)는 CI 실패.
- **D. 품질 게이트 명시화**: ko/en 대칭·blocklist 부활검사·figure 렌더·**슬롯 커버리지 리포트**를 `--check`로 CI 게이트.
- **E. 템플릿 추상화**: 반복 유형에 `spec + LANG_PACK` 베이스(이미 `i18n_poc.py`가 원형).

**우선순위: C(코드주입) &gt; A(분할) &gt; B(레지스트리) &gt; D·E.**

## 4. 지금 당신의 결정이 필요한 것

1. **통계 재설계 강도** — 정답률·평균시간을 학습자 화면에서 정말 후퇴시킬까? (당신이 이미 본 UX가 바뀜. 연구는 강하게 권함.)
2. **Elo-lite 세분 단위** — 영역별(4개) θ로 갈까, 개념별(수백개)로 갈까? (영역별이 현실적 시작점.)
3. **규모화 방향** — 천장(상위 난이도) 채우기 우선 vs 폭(새 유형) 넓히기 우선?
4. **스킵 버튼**(ROADMAP 대기 중) — "신호 있는 스킵"(다른 개념 그룹 교체+정체 카운터, 하루 3회) 설계 완료됨. 넣을까?
5. **파이프라인 재구성 시점** — 규모화 전에 지금 정리(권장) vs 나중.
