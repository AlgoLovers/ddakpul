## 검토 완료 — 사용자 결정 대기 (2026-07-24 리서치)

### 문제 교체(스킵) 버튼 — 교육적 검토 결론: 조건부 찬성
근거 리서치 요약(에이전트 2건, 상세 출처는 PR/세션 기록):
- 선택권 제공은 아동의 내재 동기·노력에 특히 효과적(Patall et al. 2008 메타분석). 단 학습 성과 자체는
  시퀀싱을 엔진이 쥘 때가 낫다(Karich 2014) → **좁은 탈출구만 제공**이 정석.
- 반례: IXL(스킵 불가+감점) = 좌절 유발로 가장 비판받는 앱. Khan Academy는 2024년 스킵 도입(스킵=오답 처리).
- 막힘의 원인이 **선수 개념 결손**일 때 같은 유형 재출제는 wheel-spinning(Beck&Gong 2013)을 연장
  → 풀이 예시 제시 또는 개념 전환이 연구가 지지하는 처치.

권장 설계("신호 있는 스킵"):
1. 채점 전에만 노출되는 "다른 문제 풀래요" 버튼 → **같은 난이도의 다른 그룹(다른 개념)** 문제로 교체.
2. 스킵 신호: 연속 정답 스트릭 끊음(정답 아님) / 연속 오답 카운트엔 불산입 /
   **건너뛴 그룹의 정체 감지(규칙 4) 카운터에 가산** → 반복 회피 시 대표문제 해설·선수 그룹 강등이 자연 발동.
3. 한도 하루 3회(초과 시 비활성+중립 카피). 처벌 연출 금지. 리포트에 스킵률 표시(회피 패턴 조기 발견).
